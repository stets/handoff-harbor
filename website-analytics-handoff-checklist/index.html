Tracking ownership needs to be boringly explicit.

Clients often assume analytics is “included” because a designer added a GA4 tag, a Meta pixel, or a contact-form conversion during the build. But if the property lives under the wrong Google account, nobody has admin access, or conversions were never tested, the launch handoff is incomplete.

The handoff record should say what is installed, who owns it, who can change it, what events are tracked, what is untracked, and whether analytics support is included after launch.

What to confirm before closing the project

Account ownership

Confirm GA4, Search Console, Tag Manager, ad pixels, heatmaps, and dashboard accounts are owned by the client or transferred cleanly.

Installed tags

Document tracking IDs, tag locations, consent/banner dependencies, form events, thank-you pages, and any third-party scripts.

Conversion proof

Test the important actions: form submit, booking click, phone/email click, checkout, newsletter signup, or quote request.

Support boundary

Define whether analytics setup, reporting, broken tags, ad-platform pixels, consent updates, or event changes are included or billable.

Copy/paste website analytics handoff checklist

  • ☐ GA4 property, measurement ID, owner account, and admin users are documented.
  • ☐ Google Search Console property is verified and the client has owner/admin access.
  • ☐ Google Tag Manager container ID and publish access are documented if GTM is used.
  • ☐ Meta/LinkedIn/TikTok/Google Ads pixels are listed, or marked not installed.
  • ☐ Cookie consent or privacy-banner dependency is documented where relevant.
  • ☐ Form submissions, booking links, phone/email clicks, purchases, or other key events are tested.
  • ☐ Thank-you pages, redirect URLs, and UTM expectations are noted.
  • ☐ Client knows where to view reports and what numbers are meaningful after launch.
  • ☐ Any untracked items are listed so the client does not assume they are included.
  • ☐ Post-launch analytics changes are tied to the warranty window, care plan, or billable support process.

Website analytics handoff record Website: [client URL] GA4 property name: [property] GA4 measurement ID: [G-XXXX] GA4 owner/admin: [client / designer / agency / email] Search Console property: [URL or domain property] Search Console owner/admin: [email] Tag Manager container: [GTM-XXXX / not used] Pixels/ad tags installed: [Meta / LinkedIn / Google Ads / none] Consent banner dependency: [yes / no / tool] Key events tested: [forms / booking / phone / email / checkout / newsletter] Report/dashboard location: [GA4 / Looker Studio / client dashboard / none] Known untracked actions: [list] Last test date: [date] Support boundary: Analytics setup, report interpretation, ad pixels, new conversion events, broken third-party tags, and consent/tracking changes are [included until date] / [covered under care plan] / [billable support].
