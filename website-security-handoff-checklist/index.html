Security gets messy when nobody owns the boring parts.

Small-business clients rarely ask about patch schedules, recovery codes, backup retention, plugin licenses, DNS access, malware alerts, or who responds if the site gets defaced. Then something breaks and the designer becomes unpaid emergency support.

The handoff should make security boring and explicit: what was configured, who owns each account, what gets monitored, how restores work, and which requests are included versus billable.

What to confirm before closing the project

Admin ownership

Confirm the client has owner/admin access, recovery email, 2FA method, billing ownership, and emergency contact details for the CMS, host, domain, and security tools.

Updates and licenses

Document theme/plugin/app update responsibilities, paid licenses, renewal dates, compatibility risks, and who approves security-related changes.

Backups and restores

Record backup location, frequency, retention, restore process, last successful restore/test, and whether emergency restore help is included.

Monitoring boundary

Clarify malware scans, uptime checks, firewall alerts, SSL/domain expiration alerts, incident response, and post-launch support costs.

Copy/paste security handoff checklist

  • ☐ Client owner/admin account exists and designer-only accounts are removed or downgraded.
  • ☐ 2FA/recovery method, recovery email, and emergency contact are confirmed.
  • ☐ Hosting, domain, DNS, CMS, security plugin, CDN, and backup tool ownership is documented.
  • ☐ SSL certificate, domain renewal, hosting renewal, and paid plugin/app renewal dates are listed.
  • ☐ Update responsibility is clear for CMS core, plugins, themes, apps, dependencies, and integrations.
  • ☐ Backup frequency, retention period, storage location, and restore steps are recorded.
  • ☐ Malware scanning, firewall, uptime, and security alert destinations are documented.
  • ☐ Unused accounts, old contractors, staging passwords, default users, and temporary access are cleaned up.
  • ☐ Known security risks or client-deferred items are written down.
  • ☐ Emergency response, malware cleanup, restore work, plugin conflicts, update labor, and monitoring changes are routed to warranty, care plan, or billable support.
