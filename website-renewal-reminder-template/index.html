The handoff is not clean until renewals are boring.

A lot of post-launch chaos is not a design problem. It is a renewal ownership problem. The client does not know which subscriptions matter, who pays for them, when they renew, or whether your maintenance plan covers them.

Send one plain renewal note while the project is still fresh. It protects the client, gives you a reason to offer paid maintenance, and makes future “the site is down” texts less likely.

What to include in the renewal note

Owned assets

Domain, DNS host, hosting account, email provider, SSL, CMS, analytics, backups, and any third-party tools tied to the site.

Renewal details

Renewal date, billing owner, approximate cost, auto-renew status, login owner, and what happens if it expires.

Support boundary

State whether renewals are client-managed, included in your maintenance plan, or handled only as paid ad hoc work.

Next action

Give the client a simple choice: confirm they own it, book a renewal review, or move into a monthly care plan.

Copy/paste renewal reminder email

Subject: Website renewal items to keep on your radar Hi [Client Name], Now that the site is launched, I want to make sure the renewal pieces are clear so nothing surprises you later. Here are the main items tied to your website: 1. Domain - Provider: [registrar] - Renewal date: [date] - Billing owner: [client / me / other] - Auto-renew: [on/off/unknown] 2. Hosting / website platform - Provider: [Webflow / WordPress host / Squarespace / other] - Renewal date: [date] - Billing owner: [client / me / other] 3. Email / DNS / SSL - Email provider: [Google Workspace / Microsoft 365 / other] - DNS host: [Cloudflare / registrar / platform] - SSL: [automatic / platform-managed / manual] 4. Paid plugins, apps, or licenses - [Tool/license]: [renewal date + owner] - [Tool/license]: [renewal date + owner] What is included after launch: - [included support, if any] What is not included unless we set up a care plan: - Monitoring renewals - Troubleshooting expired services - Plugin/license updates - New edits or feature requests - Emergency recovery work caused by missed renewals If you want me to keep an eye on these, I can set up a simple monthly care plan. Otherwise, please save this email and add the renewal dates to your calendar. Thanks, [Your Name]

Renewal handoff checklist

  • ☐ List every recurring cost connected to the website.
  • ☐ Confirm who owns billing for each item.
  • ☐ Confirm whether auto-renew is enabled and which card/account is used.
  • ☐ Add domain, hosting, email, SSL, plugin, app, and backup renewals to the client packet.
  • ☐ Include renewal dates and approximate annual/monthly costs.
  • ☐ Explain what breaks if each item expires.
  • ☐ State whether renewal monitoring is included, excluded, or part of a paid care plan.
  • ☐ Send the client one final renewal summary email before closing the project.
