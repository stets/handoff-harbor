What the login handoff should cover

Where to log in

Admin URL, platform URL, hosting panel, registrar, DNS provider, email/admin console, analytics, and support portals.

Who owns each account

Client owner/admin roles, designer/vendor roles, recovery email, billing owner, and whether your access remains active.

How access is transferred

Password manager invite, reset link, new user invitation, 2FA setup, backup codes, and confirmation that plain-text passwords are not being emailed.

What happens after handoff

State whether future login help is included, billable, covered by a care plan, or limited to a short warranty window.

Copy/paste website login handoff email

Subject: Website login handoff details Hi [Client Name], Here are the login and account ownership details for your website handoff. Website admin - Admin URL: [admin URL] - Client admin user: [name/email] - Role: [owner/admin/editor] - Password setup: [reset link sent / password manager invite / client-created password] Hosting / website platform - Provider: [Webflow / WordPress host / Squarespace / other] - Login URL: [URL] - Account owner: [client / me / shared] - Billing owner: [client / me / other] Domain and DNS - Registrar: [registrar] - DNS provider: [Cloudflare / registrar / platform] - Account owner: [client / me / other] - Recovery email: [email] Security notes - Please do not send passwords by email. - Turn on 2FA for owner/admin accounts where available. - Save backup codes somewhere safe. - If I still have contractor access, I will remove it on [date] unless we set up a care plan. After handoff, login recovery and access troubleshooting are [included until date / handled as billable support / included in your care plan]. Please confirm when you can log in successfully. Thanks, [Your Name]

Login handoff checklist

  • ☐ Create or confirm the client's owner/admin account.
  • ☐ Confirm the client can log in before removing your fallback access.
  • ☐ Record admin URLs and support URLs in the handoff packet.
  • ☐ Transfer passwords through a password manager or reset flow, not plain email.
  • ☐ Confirm recovery email, 2FA, and backup-code ownership.
  • ☐ List which contractor/vendor accounts stay active and why.
  • ☐ Set a date to remove temporary launch access.
  • ☐ State whether future login help is included, billable, or care-plan only.

Boundary line to include

Once the handoff is complete, please save these login details in your own password manager. Future access recovery, lost-password troubleshooting, or account cleanup is handled as billable support unless it is covered by an active care plan.
