Use this before you transfer or close the project

A clean access inventory does not mean emailing passwords around. It means naming every account, owner, renewal, billing responsibility, and safe password-sharing method before the project goes quiet.

If a client comes back six months later asking who owns something, this document should answer it without archaeology.

What to track

Domains and DNS

Registrar, DNS host, nameservers, important records, renewal date, billing owner, and who has admin access.

Hosting and CMS

Hosting provider, CMS admin URL, account owner, backup location, staging URL, and emergency contact path.

Tools and integrations

Analytics, forms, email marketing, booking, payments, maps, chat widgets, search, pixels, and API keys.

Licenses and renewals

Themes, plugins, fonts, stock assets, subscriptions, renewal dates, support terms, and who pays next.

Copy/paste access inventory

Account / service: [Domain registrar / hosting / CMS / plugin / analytics] Purpose: [What this account controls] Login URL: [URL only — do not paste passwords here] Account owner: [Client / designer / third party] Admin users: [Names/emails] Billing owner: [Who pays] Renewal date: [Date] Plan/license: [Plan name or license note] Password sharing method: [1Password / Bitwarden / client vault / invite sent] Recovery email / phone: [Owner-controlled recovery path] Important notes: [DNS records, plugin dependency, cancellation warning, support contact] Handoff status: [Transferred / client invited / needs client action / excluded]

Quick checklist

  • [ ] Domain registrar and DNS host identified
  • [ ] Hosting/CMS owner and billing responsibility confirmed
  • [ ] Admin users reviewed and unnecessary accounts removed
  • [ ] Plugin/theme/font/stock licenses documented
  • [ ] Analytics, forms, email, booking, and payment integrations listed
  • [ ] Passwords shared through a vault or account invitation, not plain email
  • [ ] Renewal dates and cancellation risks called out in the handoff packet
  • [ ] Anything intentionally excluded is written down so it does not become surprise free support
