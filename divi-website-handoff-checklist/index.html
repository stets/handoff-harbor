Divi projects often end with a vague “you can edit it in WordPress” handoff. That leaves the freelancer holding license details, Theme Builder logic, global presets, form routing, plugin updates, backups, and support expectations the client never agreed to own.

A Divi site is not handed off just because the client has a WordPress login.

The messy version: Divi or Elegant Themes billing stays on the freelancer's account, global presets are undocumented, Theme Builder templates are easy to break, forms point to old inboxes, backups are assumed but not tested, and every “tiny layout tweak” becomes unpaid support.

This is not legal, tax, or Elegant Themes policy advice. It is a practical project-closeout checklist for freelancers and small studios so the client receives ownership, access, editing rules, acceptance, and support boundaries in writing.

What to document before a Divi project is accepted

WordPress, hosting, and account ownership

Confirm WordPress admin ownership, hosting, registrar/DNS, SSL/CDN, backup access, recovery email, 2FA expectations, and any accounts that still live under the freelancer or agency.

Divi license, theme, and plugin renewals

Record who owns the Divi/Elegant Themes license, renewal date, connected sites, child theme status, premium plugin subscriptions, update responsibility, and what breaks if renewals lapse.

Theme Builder, global presets, and custom code

List Theme Builder templates, global colors/fonts/presets, saved layouts, reusable sections, custom CSS/JS, shortcode/plugin dependencies, and what the client should not edit casually.

Training and support boundaries

Explain safe page/text/image edits versus paid layout/template/plugin work, included bug-fix window, excluded new work, change-request process, and the care-plan option.

Copy/paste Divi handoff checklist

  • ☐ WordPress admin owner, client users, roles, recovery email, 2FA method, and contractor cleanup are confirmed.
  • ☐ Hosting, registrar, DNS, SSL, CDN/cache, email records, staging site, and deployment notes are documented.
  • ☐ Divi/Elegant Themes license owner, renewal date, connected site, theme/child-theme setup, and premium plugin licenses are listed.
  • ☐ Divi Theme Builder templates, global presets, saved layouts, custom CSS/JS, shortcode dependencies, and reusable modules are documented.
  • ☐ Forms, notification emails, SMTP/mail deliverability, spam settings, CRM/newsletter/booking/payment integrations, and test submissions are verified.
  • ☐ Analytics, pixels, Search Console, SEO plugin settings, schema, redirects, cookie/privacy tools, and conversion events are captured.
  • ☐ Backup system, restore owner, retention window, update process, Divi/theme/plugin conflict plan, and maintenance responsibility are explained.
  • ☐ Client editing notes explain safe text/image/module edits, risky Theme Builder/template/global preset changes, plugin settings, custom code, cache clearing, and image sizing.
  • ☐ Known issues, accepted tradeoffs, future recommendations, and unsupported requests are written down before final acceptance.
  • ☐ Included support window is stated, and new pages, layout work, plugin conflicts, SEO, copy, integrations, troubleshooting, or ongoing updates route to paid work.
